Peer–to–peer

Radicle works without a centralized server. No hassle with setting up your own hosting, and no need to trust companies with your data.

Terminal-first

A Radicle project contains a git repository as well as the all the issues and proposals associated with that repo. All accessible from your terminal.

Programmable

Ever wanted fine-grained control over your code collaboration service? With Radicle you each unit of functionality–a machine–is its own litte P2P program, written in the Radicle language. You can change them to suit your needs, or create entirely new ones. Resources for developing your own Radicle machines coming soon!

Collaborate with Radicle

Setup a project

rad project allows you to create, checkout, manage, and publish a project, comprised of issues, patch proposals, and a git repo. 

$ rad project init
? What's the name of your project? radicle-garden
? Briefly describe your project? A community garden

=> Adding issues to your project
=> Adding patches to your project

Manage issues

rad issue allows you to view, create, and comment on issues directly from your project directory. 

$ rad issue list
#  state   title                author   updated
1  open    Patch not accepted   janedoe  2019-01-25 13:27
0  closed  How do I contribute  randy    2019-01-25 09:14

$ rad issue comment 1 "probably the wrong size"
Added comment to issue #1

Propose changes

rad patch provides a simple system for proposing and discussing changes to a repository. 

$ rad patch list
#  state     commit                author   updated
1  pending   Make docs friendlier  janedoe  2019-01-25 13:27
0  accepted  Typo fixes            willyg   2019-01-25 09:14

$ rad patch accept 1
Merging proposal #2 with master

Radicle Stack

Each Radicle app is a peer-to-peer program that syncs with anyone who checks out your open source project.

Taking inspiration from the original git model, where a single repo maintainer accepts patches, our Radicle alpha apps are administered by the project owner -- updates can be submitted by any peer on the Radicle network.

In the coming months we’ll be sharing more about the Radicle technical stack and releasing documentation that will allow you to write your own apps using the same peer-to-peer infrastructure.
