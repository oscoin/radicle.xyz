Peer–to–peer

Open source development is a fundementally P2P process. The Radicle stack builds on IPFS to enable fully peer-to-peer conversations with code.

Terminal-first

Radicle's collaboration apps bring issue management and patch proposal tools into the terminal. Coordinate with peers and manage contributions with convenient open source utilities.

Programmable

Apps are written in the Radicle language, a Lisp dialect designed specifically for creating programs that sync across a peer-to-peer network. Resources for developing your own Radicle apps coming soon!

Collaborate with Radicle

Setup a project

rad project allows you to register your git projects on the radicle network and connect your repo to other radicle applications, like rad issues and rad diff.

Manage your issues

rad issue brings issue management to the command line. Create, view, list, and comment to heart's content.

Collaborate on code

rad diff gives you the tools to collaborate on git repositories, completely P2P. Propose and track changes to projects you follow using a patch-based collaboration model.

Radicle Stack

Each Radicle app is a peer-to-peer program that syncs with anyone who checks out your open source project.

Taking inspiration from the original git model, where a single repo maintainer accepts patches, our Radicle alpha apps are administered by the project owner -- updates can be submitted by any peer on the Radicle network.

In the coming months we’ll be sharing more about the Radicle technical stack and releasing documentation that will allow you to write your own apps using the same peer-to-peer infrastructure.
